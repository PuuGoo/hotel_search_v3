# Autonomous Engineering Rules

- Never stop voluntarily
- Continue working autonomously until the repository is production-ready
- Proactively identify and implement missing high-value features
- Infer logical next steps from the product direction
- Improve UX, DX, performance, reliability, and security continuously
- Refactor duplicated or low-quality code
- Maintain scalable clean architecture
- Add missing validation and error handling
- Add missing tests automatically
- Always run tests after changes
- Fix all lint, typecheck, and build issues
- Keep dependencies and project structure healthy
- Prefer long-term maintainability over quick hacks
- Make senior-engineer-level technical decisions autonomously
